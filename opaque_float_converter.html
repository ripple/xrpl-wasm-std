XRPL Amount Types Reference
Opaque Float Format (IOUs)
64-bit custom encoding: [Type:1][Sign:1][Exponent:8][Mantissa:54]
Type bit (bit 63): Always 1 for fungible tokens Sign bit (bit 62): 1 = positive, 0 = negative Exponent (bits 61-54): 8 bits, biased by 97 (range -96 to +80) Mantissa (bits 53-0): 54 bits providing ~16 decimal digits precision
Special Values:
  • Zero: 0x8000000000000000
  • Maximum: ~9.999999999999999 Ã— 10^80
  • Minimum positive: ~1.0 Ã— 10^-81
STAmount Format (48 bytes)
Used for IOU amounts in transactions:
Bytes 0-7: Amount (opaque float, 8 bytes) Bytes 8-27: Currency Code (20 bytes) Bytes 28-47: Issuer Account ID (20 bytes)
Currency Code Encoding
Standard 3-character codes (e.g., USD, EUR):
  • Padded with zeros to 20 bytes
  • Format: 12 zero bytes + 3 ASCII bytes + 5 zero bytes
  • Example: USD = 0x0000000000000000000000555344000000000000
Non-standard codes:
  • 40-character hex string (20 bytes)
  • Can represent any 160-bit value
Amount Types
1. XRP (Native Currency):
  • 64-bit signed integer (drops)
  • 1 XRP = 1,000,000 drops
  • Range: -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807
2. IOU (Fungible Tokens):
  • Opaque float (8 bytes) + Currency (20 bytes) + Issuer (20 bytes)
  • Total: 48 bytes (STAmount format)
  • Supports arbitrary precision decimal values
3. MPT (Multi-Purpose Tokens):
  • 64-bit unsigned integer quantity
  • Associated with MPT ID (issuance info)
